wsjcpp

Yet one... C++ Source Package Manager.

features - has implementattion

  • Creating open-source package by anyone (using github only now...)
  • Manager - one binary application
  • Support: Linux and MacOS
  • Easy install specific versions of source packages to your projects
  • Upgrade source packages
  • Hold specific version of source code - so you can fixing modules localle
  • Easy integration to you CMake project
  • Easy start new project (prepared basic struct and files)
  • System for generate files with specific classes
  • Included unit-tests systems

all targets(s) - not any implemented yet

  • fast create a little microservices based on dockerfile using c++
  • "freezing" sources codes (sometimes need for commercial development)
  • Easy publishing sources packages (github, bitbucket, gitlab, http/https servers)
  • Including helpful tool for generate specific classes (templates or snippets)
  • Including helpful tool for unit-tests
  • Including helpful tool for prepare packages to another systems: deb, brew etc...
  • Including helpful tool for docker image
  • Including helpful tool for build server
  • Including helpful tool with web interface coding, building, managment
  • Including helpful tool for fast start your local "source package registry"
  • Including helpful tool for code analizing
  • Including helpful tool for code docs

тезисы

Он же билд сервер - для проверки пакетов (на основе докера)

Без авторизации - просто ссылка на репозиторий (github) и версию пакета (типа wsjcpp publish link : version) - также сохраняются все файлы для дистрибьюции - запрещена повторная заливка пакета

С авторизацией (простенькая) - загружается сорс код по файлово - то что в дистрибьюции - надо проработать протокол типа заливается вначале wsjcpp.yml - потом дозаливаются все остальные файлы

Регистрация только в ручном режиме - заходим в админку и добавляем пользователя с ролью девелопер

Состояние сборки и тестов (если не прошли тесты и сборка то пакет дропается с сервера а состояния остаются)

Хранятся последняя версия пакета а также версии на которые ссылаются предыдущие + хранятся unit-tests - весь остальной храм удаляется

Команды:

wsjcpp publish ссылка-на-гитхаб:бранч-или-тег

wsjcpp publish путь-до-локального-пакета

wsjcpp publish --origin ссылка-сервера-регистратора что-куда

если оригин не указан то используется дефолтный wsjcpp.org
